# Security

## Reporting a vulnerability

Please report security issues **privately** so we can address them before they are widely known.

**Do not** open a public GitHub issue for undisclosed security vulnerabilities.

### Contact

**Dieter Wellerdiek** — [dieter@los-aemmerles.org](mailto:dieter@los-aemmerles.org)

Include as much detail as you can: affected component, steps to reproduce, impact, and suggested fix if you have one. We will acknowledge receipt when possible; response times depend on volunteer availability.
